# ContextBar v0.3.23

## Added

- **Cost tab** — a new tab in the menubar app that estimates what your Claude Code and Codex usage *would* cost on the metered API, per day and per project. It's the native equivalent of `better-ccusage daily --instances`: each day lists the projects you worked in with their input/output tokens and estimated cost. You're on a subscription, so these are estimates — not bills.
- **Plan-value projection** — projects a monthly metered-API cost from the last 30 days and compares it to your actual plan price (e.g. *"≈ $8,302/mo — about 42× your Max 20× plan"*). No other tracker reframes API-equivalent cost against the plan you actually pay for.
- **Cache-savings insight** — shows the net dollars prompt caching saved versus paying full input price (e.g. *"Prompt caching saved $47,139 in the last 30 days"*).
- **30-day cost-trend sparkline** with peak $/day, for at-a-glance burn awareness.

## How costs are computed

Each turn is priced by model from the LiteLLM rate table — the same canonical source `ccusage` uses — fetched live with a 24h on-disk cache (`~/.context-bar/pricing.cache.json`) and a bundled offline fallback. The estimate sums `input·input_rate + output·output_rate + cache_creation·cache_write_rate + cache_read·cache_read_rate`, honoring Anthropic's prompt-cache multipliers and the >200K long-context tier where a model carries one. A precomputed `costUSD` on a transcript turn is used when present (ccusage "auto" mode). Set `CONTEXTBAR_PRICING_OFFLINE=1` to skip the live rate fetch.

These numbers also appear in the standalone `~/.context-bar/detail.html` export, and the snapshot now carries the per-bucket token split (input / output / cache create / cache read) and estimated cost across the daily / weekly / monthly / model / project / session aggregates.
