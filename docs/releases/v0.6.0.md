# ContextBar v0.6.0

## Added

- **Live 5h-block dashboard — `context-bar blocks` & `context-bar live`.** See your active 5-hour window burn at a glance: usage % of limit (color-tiered gauge), burn rate ($/hr · tok/min), projected block total, ETA-to-limit, and a reset countdown. `blocks` prints once; `live` is an auto-refreshing terminal dashboard (`ratatui`, `--interval`, `q` to quit, `r` to refresh). Cross-platform — runs on macOS, Linux, Windows, and over SSH.
- **Monthly budget + menubar color (macOS app).** Set a monthly budget in **General** settings and the menubar title shows a yellow→red dot as you near it — using the worse of your 30-day run-rate vs the budget and your hottest 5-hour limit %. Leave it at 0 to keep just the 5h-limit tint.
- **Install it anywhere — `cargo install context-bar` and `npx context-bar`.** Tagged releases now publish to crates.io and npm (six per-platform prebuilt binaries + a tiny launcher, no postinstall), alongside the existing macOS DMG/Homebrew cask and the downloadable binaries for macOS / Linux (musl) / Windows.

## Changed

- The block views are powered by a reusable `core::live::block_status`, ready for a native popover gauge next.

## For maintainers

- `cargo install` / `npx` publishing is gated on the `CARGO_REGISTRY_TOKEN` and `NPM_TOKEN` repo secrets (one-time setup; see `docs/PUBLISHING.md`). Until they're set, the publish jobs no-op and the DMG + binary release ship as before.
