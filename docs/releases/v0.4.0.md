# ContextBar v0.4.0

## Added

- **Now in your terminal — `context-bar daily / weekly / monthly / session`.** ContextBar's cost engine ships as a CLI. Run a verb and get a ccusage-style table right in your shell — **Date · Agent · Models · Input · Output · Cache Create · Cache Read · Total · Cost** — grouped by period. Use `--instances` to split by project (per day × project), `--breakdown` for a per-model table, `--agent` to filter, `--since` / `--until` for a date range, `--json` for scripting, `--offline` to skip the live pricing fetch, `--lang en|tr` for bilingual output, and `--no-color`. Same engine as the menubar app, so **Total** is ccusage's "Total Tokens" (all four buckets) — not the Stats tab's fresh-work total.
- **`cargo install` just works.** The engine is self-contained: `usage_signal.py` is embedded and materialized to `~/.context-bar` on first run, so there's no separate checkout to wire up.

## Changed

- **Cleaner internals — `context-bar-core`.** The engine, cost/token math, bilingual i18n, and the new report module moved into a cross-platform `context-bar-core` crate; the menubar app, the CLI, and the Zed extension all re-export it. This also fixed a latent wasm32 build break in the Zed extension.
- Cost figures are still API-equivalent estimates, not a bill — the model fidelity carries over unchanged from the menubar.
