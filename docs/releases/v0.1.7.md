# ContextBar v0.1.7

Packaging metadata fix.

## Highlights

- **Correct version display in local builds**: `dist/ContextBar.app` now picks up the real app version and build number during local packaging, instead of showing the template `0.1.0 (1)`.
- **Consistent About screen**: the version shown in the app now matches the actual release and installed bundle, regardless of whether the app was opened from `/Applications` or from a local `dist` build.
